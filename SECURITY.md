# Security Policy

## Reporting a Vulnerability
If you discover a vulnerability in this project, please follow these steps:

1. **Do not open a public issue.**
2. Contact us directly at **matveidoroshenko@gmail.com**.
3. Alternatively, use the [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories) feature to report privately.

We will:
- Acknowledge your report within **48 hours**.
- Provide a status update within **5 business days**.
- Work with you to validate, patch, and disclose the issue responsibly.

## Security Best Practices
To help keep the project secure:
- Always use the latest stable release of SFML and dependencies.
- Avoid using outdated libraries or unverified third-party code.
- Run static analysis tools (e.g., `clang-tidy`, `cppcheck`) to detect common issues.
- Follow secure coding guidelines (e.g., input validation, memory safety).

## Responsible Disclosure
We kindly ask that you:
- Give us reasonable time to fix the issue before public disclosure.
- Avoid exploiting vulnerabilities in production environments.
- Share detailed steps so we can reproduce the issue quickly.
